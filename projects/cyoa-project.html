CYOA project

Context

The instructions of this project were to create an adventure type videogame that allowed the user to make specific choices and decisions through the use of creating and linking files that when clicked on will send the user to specific sections of the videogame. The purpose of this project is meant to help in developing skills with not only the command line but also with using markdown to create text and files with links. It's also meant to help in improving communication skills and in problem solving skills within our IDE.

title of project & prompt/instructions

Our partner and I had the idea of making a game where there's only 60 seconds until the world ends and the user would be given a set of options that lead to different endings or areas within the game. The rubric however, gave us a list of requirements that were needed in order to get full credit.

  1. have a google drawing plan
  2. create at least ten .md files
  3. create at least 1 subdirectory/sub file
  4. create functional links that take you to different files
  5. add a README.md file and include a start button aswell as usernames for credits and a link to our sketch
  6. complete the partner evaluation after completing the cyoa project

Content/what was made

My partner and I first started the project by planning out our draft by setting each option into seperate files depending on the option that was chosen. Then we split roles by colorcoding each option that can be chosen so that we could equally complete each task. So, if the user were to do a specific action such as going to another area then they will be sent to another section/file and they can continue untill they get a game over or until the game is complete. So for my section I decided to create two files. One being a get-ready.md file and a decision.md file that would allow the user to make multiple options and decisions rather then only have one specific file that makes the user only have a few options. After we had finished the rough draft, we each started working on the project and eventually we had finished.

Reflection

challenges

The main challenge that I had really faced was in linking files that were in multiple subdirectories. An example of this was when I had been organizing specific .md files within a subdirectory called new-area which included all the options that were made when in the new area. I was testing out to make sure that the files were working when I had gotten an error when I clicked on the "start over" link. The error said that the file/directory could not be found. I was confused as to why it wasn’t working because everything looked the same as to my other files but I realized that it may had been because since I had organized my files within multiple subdirectories, I needed to include the .. command when linking files that were within multiple subdirectories and when I tried again the problem was fixed!

code before:

[start over](../alert.md) --> ERROR

code after:

[start over](../../alert.md) --> all good!

Takeaways

Overall what I learned from this project is the importance of communication and teamwork. I had experienced a lot of issues with the weather which had negatively impacted my ability to go to school. However, because of my clear communication and the perseverance I had I still was able to work on the project even when I was at home and unable to see my partner in person. Since I was able to maintain clear communication with my partner we solved the errors that occurred when working on the project and we completed the project with ease.

Next Steps

Next time I feel like I should have added a button that allows the user to go back to the other file they were in because it would be alot more efficient for the user to be able to go back and make another decision rather then just start all over again. Overall, I had a great expirience with this project and despite the challenges I faced I was able to improve my skills with linking files and using my IDE. I hope I have more projects to share soon!
