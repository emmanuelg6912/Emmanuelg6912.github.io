CYOA project

Context

The instructions of this project were to create an adventure type videogame that allowed the user to make specific choices and decisions through the use of creating and linking files that when clicked on will send the user to specific sections of the videogame. The purpose of this project is meant to help in developing skills with not only the command line but also with using markdown to create text and files with links. Its also meant to help in improving communication skills and in problem solving skills within our IDE

title of project

our partner and I had the idea of making a game where theres only 60 seconds until the world ends and the user would be given a set of options that lead to different endings or areas within the game. The rubric however, gave us a list of requirements that were needed in order to get full credit
  1. have a google drawing plan
  2. create at least ten .md files
  3. create at least 1 subdirectory/sub file
  4. create functional links that take you to different files
  5. add a README.md file and include a start button aswell as usernames for credits and a link to our sketch
  6. complete the partner evaluation after completing the cyoa project

Content/what was made

My partner and I first started the project by planning out our draft by setting each option into seperate files depending on the option that was chosen. Then we split roles by colorcoding each option that can be chosen so that we could equally complete each task. So if the user were to do a specific action such as going to another area then they will be sent to another section/file and they can continue untill they get a game over or until the game is complete.
